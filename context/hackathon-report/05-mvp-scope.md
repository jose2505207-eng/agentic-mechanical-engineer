# MVP Scope Boundaries (Explicit Non-Goals)

## IN
- One robot class (4-wheel mobile inspection robot)
- Text input
- Template-based CAD
- Static + quasi-static checks
- Rule-based failure prediction
- Curated BOM
- STL export
- Web UI

## OUT
- Arbitrary robot types
- Image/CAD upload input
- FEA (no stress fields — analytical checks only)
- Dynamic gait/motion planning
- Injection molding/CNC analysis
- Digital twin beyond the PyBullet scene
- Multi-agent design review panel
