# Project Overview

**Source:** MECHANICAL ENGINEER AGENT — Master Context Baseline v1.0

Mechanical Engineer Agent (MEA) is an AI-native engineering platform that functions as an AI Mechanical Engineer capable of transforming requirements, natural language instructions, CAD models, existing components, and engineering constraints into manufacturable designs, simulations, optimization recommendations, and production-ready outputs.

The long-term vision is not to build another CAD platform. The vision is to become the **"Copilot for Mechanical Engineering."**

Users should describe what they want to build and receive a complete engineering package.

The platform should eventually act as:
- Mechanical Engineer
- Robotics Engineer
- Manufacturing Engineer
- Systems Engineer
- Design Engineer
- Materials Engineer
- Simulation Engineer

The goal is to move engineering from **Engineer → Tools** to **Engineer → AI Engineer**.

## Core Mission

Enable engineers, manufacturers, researchers, startups, and makers to move from an idea to a manufacturable product through conversation.

**Input:** "Build me a machine that does X."

**Output:**
- Requirements
- Architecture
- CAD
- Material Selection
- Simulation
- Validation
- Optimization
- Manufacturing Package

## Core Differentiation

Most competitors do: `Prompt → CAD` or `Prompt → 3D Model`

Mechanical Engineer Agent does:

```
Prompt
↓ Requirements Engineering
↓ System Architecture
↓ Component Selection
↓ Material Selection
↓ CAD Generation
↓ Digital Twin Creation
↓ Simulation
↓ Failure Prediction
↓ Optimization
↓ Manufacturing Validation
↓ Cost Analysis
↓ BOM
↓ Production Package
```

**The system generates engineering decisions rather than geometry alone.**
